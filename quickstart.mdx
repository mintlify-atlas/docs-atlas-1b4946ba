---
title: "Quickstart"
description: "Get AdVerify running in your Android app in under 10 minutes"
---

<Steps>
  <Step title="Deploy the server">
    Clone the repository and install dependencies in the `server/` directory.

    ```bash
    cd server
    bun install
    ```

    Create a `.env` file and set your PostgreSQL connection string:

    ```bash
    DATABASE_URL="postgresql://user:password@localhost:5432/adverify"
    ```

    Push the schema to your database and start the development server:

    ```bash
    bun run db:push
    bun run dev
    ```

    The server starts on `http://localhost:3000` by default. You should see:

    ```
    AdVerify Server running on http://localhost:3000
    ```

    <Note>
      For production, run `bun run build` then `bun run start` to use the compiled
      output from `dist/app.js`.
    </Note>
  </Step>

  <Step title="Create an API key">
    Open the admin panel at `http://localhost:3000` (or your deployed URL) and
    navigate to **Keys**.

    Click **Create API Key**, enter your app name and Android package name, then
    copy the generated key — you'll need it in the next step.

    <Tip>
      Each API key maps to one Android app. Create a separate key per package
      name so you can track impressions, clicks, and PIN verifications
      independently.
    </Tip>
  </Step>

  <Step title="Add the SDK to your Android project">
    Copy `adverify.aar` into your app's `libs/` directory, then add the
    dependency in your module-level `build.gradle`:

    ```groovy
    dependencies {
        implementation fileTree(dir: 'libs', include: ['*.aar'])
    }
    ```

    Sync your Gradle project. No additional dependencies are required — the SDK
    has zero external dependencies and targets a minimum API level of 21.

    <Note>
      The SDK uses only Android framework APIs (`Canvas`, `Paint`, `HttpURLConnection`,
      `Settings.Secure`) and ships no third-party libraries.
    </Note>
  </Step>

  <Step title="Initialize in your Application class">
    Call `AdVerify.init()` once in `Application.onCreate()` before any activity
    is created:

    ```java
    import android.app.Application;
    import com.adverify.sdk.AdVerify;

    public class MyApp extends Application {

        @Override
        public void onCreate() {
            super.onCreate();
            AdVerify.init(
                this,
                "YOUR_API_KEY",          // key from the admin panel
                "https://your-server.com" // base URL of your deployed server
            );
        }
    }
    ```

    Register `MyApp` in your `AndroidManifest.xml`:

    ```xml
    <application
        android:name=".MyApp"
        ... >
    ```

    `init()` validates the API key and base URL at call time and stores the
    device's `ANDROID_ID` as the device identifier. It throws
    `IllegalArgumentException` if either argument is null or empty.

    <Warning>
      Do not call `AdVerify.show()` before `AdVerify.init()`. Doing so throws
      `IllegalStateException: AdVerify.init() must be called before show()`.
    </Warning>
  </Step>

  <Step title="Show ads in your Activity">
    Call `AdVerify.show()` from any `Activity` where you want to display the
    ad (or PIN verification) flow:

    ```java
    import com.adverify.sdk.AdVerify;
    import com.adverify.sdk.AdVerifyCallback;

    public class MainActivity extends AppCompatActivity {

        @Override
        protected void onResume() {
            super.onResume();
            AdVerify.show(this, new AdVerifyCallback() {
                @Override
                public void onAdShown() {
                    // Ad dialog is visible
                }

                @Override
                public void onAdClosed() {
                    // User dismissed the ad
                }

                @Override
                public void onAdClicked(String redirectUrl) {
                    // User tapped the ad's action button
                }

                @Override
                public void onError(String message) {
                    // Handle errors (no ads, network failure, max PIN attempts, etc.)
                }
            });
        }
    }
    ```

    `show()` is safe to call even if no ads are configured — the callback
    receives `onError("No ads available")` and the UI is left unchanged.

    If you don't need lifecycle callbacks, use the shorter overload:

    ```java
    AdVerify.show(this);
    ```
  </Step>
</Steps>

## What happens next

After `AdVerify.show()` is called, the SDK runs the following flow automatically:

1. **`/api/sdk/init`** — the SDK posts the device ID to the server. The response indicates whether PIN verification is required.
2. **PIN flow** (if `pinEnabled=true` and `pinVerified=false`) — a `PinVerifyDialog` is displayed. The user taps **Generate PIN & Verify**, which opens a shortener link in the browser. After completing the verification article/ad, the user receives a PIN and enters it in the dialog.
3. **Ad flow** — once the device is verified (or PIN is disabled), the SDK fetches ads from `/api/sdk/ads` and shows the first active ad in an `AdDialog`.
