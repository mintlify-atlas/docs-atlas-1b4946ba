---
title: "API Keys"
description: "Create, list, update, and delete API keys that grant SDK clients access to the AdVerify platform."
---

API keys identify an application (by `appName` and optional `packageName`) and gate access for SDK clients. Each key receives its own PIN config record upon creation. Deleting a key cascades to all associated ads, impressions, clicks, PIN config, and user PINs.

**Authentication**: HTTP Basic Auth required on all endpoints — see [Authentication](/api-reference/admin/auth).

---

## GET /api/admin/keys

Returns all API keys ordered by creation date descending.

### Request

No parameters.

### Response

An array of API key objects.

<ResponseField name="id" type="integer" required>
  Unique numeric identifier for the key.
</ResponseField>

<ResponseField name="key" type="string" required>
  The 32-character hex string that SDK clients send in the `x-api-key` header.
</ResponseField>

<ResponseField name="appName" type="string" required>
  Human-readable name for the application this key belongs to.
</ResponseField>

<ResponseField name="packageName" type="string" required>
  Android package name (e.g. `com.example.app`). Empty string if not set.
</ResponseField>

<ResponseField name="isActive" type="boolean" required>
  Whether this key is currently accepted by the SDK auth middleware. Inactive keys return 403.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of when the key was created.
</ResponseField>

### Example

```bash
curl -X GET https://your-server.com/api/admin/keys \
  -H "Authorization: Basic YWRtaW46eW91cnBhc3N3b3Jk"
```

**200 OK**

```json
[
  {
    "id": 3,
    "key": "a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4",
    "appName": "My Android App",
    "packageName": "com.example.myapp",
    "isActive": true,
    "createdAt": "2025-11-01T09:14:22.000Z"
  }
]
```

## POST /api/admin/keys

Creates a new API key. A 32-character hex key is generated automatically using `crypto.randomBytes`. A default PIN config record is also created for the key in the same transaction.

### Request

<ParamField body="appName" type="string" required>
  Human-readable name for the application. Minimum 1 character.
</ParamField>

<ParamField body="packageName" type="string">
  Android package name. Optional — defaults to an empty string if omitted.
</ParamField>

### Response

The newly created API key object (same shape as items in the list response).

### Example

```bash
curl -X POST https://your-server.com/api/admin/keys \
  -H "Authorization: Basic YWRtaW46eW91cnBhc3N3b3Jk" \
  -H "Content-Type: application/json" \
  -d '{"appName": "My Android App", "packageName": "com.example.myapp"}'
```

**200 OK**

```json
{
  "id": 4,
  "key": "f7e8d9c0b1a2f7e8d9c0b1a2f7e8d9c0",
  "appName": "My Android App",
  "packageName": "com.example.myapp",
  "isActive": true,
  "createdAt": "2025-11-15T13:45:00.000Z"
}
```

## PATCH /api/admin/keys/:id

Partially updates an existing API key. Only fields included in the request body are modified.

### Path Parameters

<ParamField path="id" type="integer" required>
  The numeric ID of the API key to update.
</ParamField>

### Request

<ParamField body="appName" type="string">
  New display name for the application.
</ParamField>

<ParamField body="packageName" type="string">
  New Android package name.
</ParamField>

<ParamField body="isActive" type="boolean">
  Set to `false` to deactivate the key (SDK clients using it will receive 403). Set to `true` to reactivate.
</ParamField>

### Response

The updated API key object.

### Example

```bash
curl -X PATCH https://your-server.com/api/admin/keys/4 \
  -H "Authorization: Basic YWRtaW46eW91cnBhc3N3b3Jk" \
  -H "Content-Type: application/json" \
  -d '{"isActive": false}'
```

**200 OK**

```json
{
  "id": 4,
  "key": "f7e8d9c0b1a2f7e8d9c0b1a2f7e8d9c0",
  "appName": "My Android App",
  "packageName": "com.example.myapp",
  "isActive": false,
  "createdAt": "2025-11-15T13:45:00.000Z"
}
```

## DELETE /api/admin/keys/:id

Permanently deletes an API key. This operation cascades — all ads, impressions, clicks, the PIN config, and all user PIN records associated with this key are also deleted and cannot be recovered.

### Path Parameters

<ParamField path="id" type="integer" required>
  The numeric ID of the API key to delete.
</ParamField>

### Request

No request body.

### Response

<ResponseField name="status" type="string" required>
  Always `"deleted"` on success.
</ResponseField>

### Example

```bash
curl -X DELETE https://your-server.com/api/admin/keys/4 \
  -H "Authorization: Basic YWRtaW46eW91cnBhc3N3b3Jk"
```

**200 OK**

```json
{ "status": "deleted" }
```
