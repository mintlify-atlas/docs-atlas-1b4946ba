Ads are the creatives served to Android devices by the SDK. Each ad belongs to one API key (app), and the SDK fetches active ads for the matching key via `GET /api/sdk/ads`.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `id` | `Int` | auto | Auto-incremented primary key |
| `apiKeyId` | `Int` | â€” | Foreign key linking the ad to an API key (app) |
| `title` | `String` | â€” | Ad headline shown in the dialog |

| `imageUrl` | `String` | `""` | URL of the banner/background image. Loaded by the SDK via `HttpURLConnection` |
| `redirectUrl` | `String` | `""` | URL opened when the user taps the action button |
| `adType` | `String` | `"interstitial"` | Display format: `interstitial`, `banner`, or `dialog` |
| `buttonText` | `String` | `"Visit Now"` | Label on the action button |
| `priority` | `Int` | `0` | Sort order: higher numbers are served first |
| `isActive` | `Boolean` | `true` | Inactive ads are excluded from the SDK feed |
| `createdAt` | `DateTime` | now | Timestamp of ad creation |

## Ad types

| Value | Behaviour |
|-------|-----------|
| `interstitial` | Full-screen dialog that appears between app interactions |
| `banner` | Compact strip shown inline within app content |
| `dialog` | Modal dialog (similar to interstitial but can be triggered manually) |

## Priority ordering

When the SDK requests ads, the server returns all active ads for the app ordered by `priority` descending. An ad with `priority = 10` is always served before an ad with `priority = 1`. Ads with equal priority are ordered by `createdAt` descending.

## Tracking impressions and clicks

Every time the SDK renders an ad it sends `POST /api/sdk/impression`. Every time a user taps the action button it sends `POST /api/sdk/click`. The admin panel shows the cumulative counts for each ad in the **Impressions** and **Clicks** columns.

These counts feed into the overall CTR displayed on the dashboard.

## Enabling and disabling an ad

Click **Disable** or **Enable** in the ad row. This sends `PATCH /api/admin/ads/:id` with `{ "isActive": false/true }`. Disabled ads are excluded from the SDK feed immediately.

## Deleting an ad

Click **Delete** on the ad row. This sends `DELETE /api/admin/ads/:id` and permanently removes the ad and all its impression and click records.

## API reference

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/admin/ads` | List all ads (includes impression/click counts and app name) |
| `POST` | `/api/admin/ads` | Create a new ad |
| `PATCH` | `/api/admin/ads/:id` | Update any ad field |
| `DELETE` | `/api/admin/ads/:id` | Delete an ad and all its tracking data |
